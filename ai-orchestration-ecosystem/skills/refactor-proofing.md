# Skill: Refactor Proofing

## When To Use

Use this when the task benefits from Refactor Proofing.

## Procedure

1. Define expected behavior.
2. Identify risks and invariants.
3. Execute in small steps.
4. Verify results.
5. Document evidence and residual risks.
6. Create or update a refactor safety contract before medium or high-risk refactors.
7. Update the future-change guide when architecture, design, UX, public contracts, data flow, or verification strategy changes.
