# Versioning

## Purpose

Maintain Sagaz as a reliable, safe, versionable, low-token orchestration ecosystem.

## Checklist

- Is this reusable?
- Does it reduce future risk?
- Does it preserve low token usage?
- Is the user impact clear?

## Release Versioning Gate

Use `protocols/release-versioning-gate.md` before any Sagaz version bump, Git tag, GitHub release, or npm publish.

Version bumps must match impact:

- Patch: compatible fixes or clarifications.
- Minor: compatible new capabilities.
- Major: breaking contract, layout, manifest schema, installer, invocation, or runtime baseline changes.
